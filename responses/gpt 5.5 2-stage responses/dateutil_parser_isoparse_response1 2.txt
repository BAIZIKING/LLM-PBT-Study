1. For any supported valid ISO-8601 input, the result is a `datetime.datetime` whose date/time fields represent the date and time encoded by the input.

2. When components are omitted, the returned `datetime.datetime` uses the lowest valid default values: missing month/day default to `1`, and missing time components default to `00:00:00.000000`.

3. Equivalent extended and compact representations parse to equal datetimes, e.g. `YYYY-MM-DD` and `YYYYMMDD`, or `hh:mm:ss` and `hhmmss`.

4. Time zone outputs match the encoded offset: `Z`, `+00:00`, `+0000`, and `+00` produce UTC `tzinfo`, while nonzero offsets produce a `tzinfo` whose UTC offset equals the specified offset.

5. Fractional seconds using either `.` or `,` produce the same microsecond value, with 1–6 fractional digits interpreted as decimal fractions of a second.
