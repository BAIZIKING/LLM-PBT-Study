1. The result is always an integer in the unsigned 32-bit range: `0 <= result <= 2**32 - 1`.

2. For the same `data` and same starting `value`, the output is deterministic and always identical.

3. Calling `zlib.adler32(data)` is equivalent to calling `zlib.adler32(data, 1)`.

4. The checksum can be computed incrementally: for any `a`, `b`, and starting `value`, `zlib.adler32(a + b, value)` equals `zlib.adler32(b, zlib.adler32(a, value))`.

5. For empty input data, the checksum is the starting value interpreted as an unsigned 32-bit Adler-32 value; in particular, `zlib.adler32(b"")` returns `1`.
