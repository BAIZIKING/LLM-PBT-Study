1. The returned value is always either `default` or a node in `G`.

2. If the returned value is not `default`, it is a common ancestor of `node1` and `node2`: there is a directed path from the returned node to each of `node1` and `node2`, allowing a node to be its own ancestor.

3. If the returned value is `default`, then `node1` and `node2` have no common ancestor in `G`.

4. If the returned value is not `default`, it is lowest among the common ancestors: none of its descendants is also a common ancestor of `node1` and `node2`.

5. The result is symmetric in the two query nodes: `lowest_common_ancestor(G, node1, node2, default)` returns the same value as `lowest_common_ancestor(G, node2, node1, default)`.
