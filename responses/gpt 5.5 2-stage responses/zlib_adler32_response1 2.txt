1. The return value is always an integer in the unsigned 32-bit range: `0 <= result <= 0xFFFFFFFF`.

2. For the same `data` and starting `value`, the function is deterministic and always returns the same checksum.

3. Omitting `value` is equivalent to passing `value=1`.

4. For empty `data`, the result is the starting checksum value, i.e. `zlib.adler32(b"", value)` returns `value` as an unsigned 32-bit checksum.

5. Checksums can be computed incrementally: `zlib.adler32(data1 + data2, value)` equals `zlib.adler32(data2, zlib.adler32(data1, value))`.
