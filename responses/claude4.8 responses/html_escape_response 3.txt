## Generation Strategy

To exercise `html.escape` thoroughly, I want my input strings to contain a rich mix of:

1. **The special characters themselves** (`&`, `<`, `>`, `"`, `'`) — these are the characters being translated, so they must appear frequently.
2. **Ordinary text characters** — letters, digits, whitespace, that should pass through untouched.
3. **Unicode characters** — to ensure non-ASCII text is unaffected.
4. **Edge cases** — empty strings, strings with only special characters, repeated specials.

I achieve this by building strings from an alphabet that heavily weights the five special characters alongside general Unicode text, using `st.text` with a custom alphabet plus plain `st.text()` mixed together. I also generate the `quote` flag as a random boolean (including its default-equivalent `True`).

## Properties to Check

Based on the documentation:

1. **`&` is always escaped** to `&amp;`. Since other escapes (`&lt;`, `&gt;`, `&quot;`, `&#x27;`) all begin with `&` but `escape` translates `&` *first*, the output must contain no raw `&` that isn't part of a valid entity — simplest invariant: the raw special chars `<` and `>` never appear in the output.
2. When `quote=True`, raw `"` and `'` must not appear in the output either.
3. **Idempotence of removal**: every special char from the input is gone from the output (replaced). Conversely, escaping a string with no specials returns it unchanged.
4. **Correctness via reference**: the output equals the input with the documented replacements applied in order (`&` first, then `<`, `>`, and optionally `"`, `'`).

The strongest single check is the reference-implementation comparison, which subsumes the others.
