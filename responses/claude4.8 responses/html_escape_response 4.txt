## Generation Strategy

To generate a wide variety of inputs and edge cases for `html.escape`, I'll use the following approach:

1. **Use `st.text()`** as the base strategy to generate random Unicode strings, which covers letters, digits, symbols, whitespace, and exotic Unicode characters.

2. **Inject special characters explicitly**: The function specifically handles `&`, `<`, `>`, `"`, and `'`. I'll create a strategy that draws characters from an alphabet that heavily includes these special characters to ensure they appear frequently (not just by random chance).

3. **Include edge cases**: Empty strings, strings of only special characters, and strings mixing special and normal characters. Using `st.text()` with a custom alphabet that combines special characters and printable characters ensures good coverage.

4. **Test both `quote=True` and `quote=False`** using a boolean strategy, since the behavior differs based on this flag.

## Properties to Check

Based on the API documentation:

1. **`&` is always escaped** to `&amp;` — regardless of the `quote` flag.
2. **`<` is always escaped** to `&lt;` — regardless of the `quote` flag.
3. **`>` is always escaped** to `&gt;` — regardless of the `quote` flag.
4. **`"` and `'` are escaped only when `quote=True`** (to `&quot;` and `&#x27;` respectively); when `quote=False`, they should remain unchanged.
5. **No raw `&`, `<`, `>` remain** in the output (they must all be converted). When `quote=True`, no raw `"` or `'` should remain either.
6. **Ampersand correctness**: Since `&` is escaped first, the output should not contain corrupted/double-escaped sequences. We verify by counting occurrences of each special char in input matches expected escaped sequences.
