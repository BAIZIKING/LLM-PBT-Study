## Strategy for Generating Inputs

To generate a wide variety of inputs and edge cases for `html.escape`, I'll use the following approach:

1. **Use `st.text()`** as the base strategy to generate random Unicode strings. This naturally covers empty strings, ASCII text, Unicode characters, and arbitrary content.

2. **Bias toward special characters**: Since `html.escape` specifically handles `&`, `<`, `>`, `"`, and `'`, I'll use an alphabet that includes these characters more frequently, ensuring edge cases involving multiple/repeated special characters are well-exercised.

3. **Combine strategies** using `st.one_of()` to mix general text with text heavily containing special characters, plus edge cases like empty strings.

4. **Generate the `quote` parameter** as a boolean (`st.booleans()`) to test both `quote=True` and `quote=False` branches.

## Properties to Check

Based on the API documentation:

1. **`&`, `<`, `>` are always escaped** (regardless of `quote`): The output must not contain raw `<` or `>`, and `&` only appears as part of an entity (`&amp;`, `&lt;`, etc.).
2. **Correct replacements**: `&`ŌåÆ`&amp;`, `<`ŌåÆ`&lt;`, `>`ŌåÆ`&gt;`.
3. **`quote` behavior**: When `quote=True`, `"`ŌåÆ`&quot;` and `'`ŌåÆ`&#x27;`. When `quote=False`, quotes are left unchanged.
4. **Idempotence is NOT expected** (because `&` gets re-escaped), so we verify replacement counts via reconstruction instead.
5. **Output is a string**, and escaping `&` first prevents double-escaping issues ŌĆö verifiable by checking that unescaping back recovers the original.
