## Strategy Explanation

**Generation Strategy:**

To generate a wide variety of inputs and edge cases for `html.escape`, I'll use `st.text()` which generates random Unicode strings. To ensure we hit the special characters that the function specifically handles (`&`, `<`, `>`, `"`, `'`), I'll create a custom alphabet that mixes these special characters with regular printable characters and arbitrary Unicode. This way, the generated strings will frequently contain the characters of interest in various combinations, while also testing normal text, empty strings, and Unicode edge cases. I'll also generate the `quote` boolean parameter to test both code paths.

**Properties to Check:**

Based on the API documentation:

1. **Always-escaped characters:** The characters `&`, `<`, and `>` should never appear in their raw form in the output (they're always converted to `&amp;`, `&lt;`, `&gt;`).
2. **Conditional quote escaping:** When `quote=True`, the characters `"` and `'` should also be escaped (converted to `&quot;` and `&#x27;`); when `quote=False`, they should remain in the output if present.
3. **Correctness of substitutions:** The output should equal a manually-computed reference escaping (`&` first to avoid double-escaping, then the others), confirming the exact replacement behavior.
4. **Output is a string:** The result is always a `str`.
