## Generation Strategy

To generate a wide variety of inputs and edge cases for `Decimal.from_float`, I will use a combination of strategies:

1. **`st.floats()`** with `allow_nan=True` and `allow_infinity=True` to cover normal floats, NaN, positive/negative infinity, subnormals, very large/small magnitudes, and zeros (including negative zero).
2. **`st.integers()`** to cover the `int` input case, including very large integers (since `Decimal.from_float` accepts both `float` and `int`).

I combine these with `st.one_of()` so each test run picks either a float or an int.

## Properties to Check

Based on the API documentation:

1. **Type guarantee**: The result is always a `Decimal` instance.
2. **NaN handling**: `Decimal.from_float(float('nan'))` should produce a NaN `Decimal` (checkable via `.is_nan()`).
3. **Infinity handling**: `Decimal.from_float(float('inf'))` / `(float('-inf'))` should produce infinite `Decimal`s (`.is_infinite()`), with the sign preserved.
4. **Exact value equivalence**: For finite floats/ints, the resulting `Decimal` must be numerically equal to the original value (since `from_float` stores the exact representable value, converting it back to `float` must yield the same float; for ints the Decimal must equal the int).
5. **Sign / zero correctness**: A finite input's resulting Decimal should compare equal to the input numerically.
