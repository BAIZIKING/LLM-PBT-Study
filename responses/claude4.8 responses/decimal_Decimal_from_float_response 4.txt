## Generation Strategy

To generate a wide variety of inputs and edge cases for `Decimal.from_float(f)`, I'll use a combination of strategies:

1. **Floats**: Use `st.floats()` to cover regular floats, including subnormals, very large/small magnitudes, positive/negative zero, NaN, and infinities (by allowing `allow_nan=True` and `allow_infinity=True`).
2. **Integers**: Use `st.integers()` since the API explicitly accepts `int` too, including large integers and negative values.
3. Combine these with `st.one_of(...)` so both accepted types are exercised.

## Properties to Check

Based on the API documentation:

1. **Return type**: The result is always a `Decimal` instance.
2. **NaN handling**: `from_float(float('nan'))` returns `Decimal('NaN')` — the result should be NaN.
3. **Infinity handling**: `from_float(float('inf'))` / `from_float(float('-inf'))` should return infinite Decimals with the correct sign.
4. **Exact representation**: For finite floats/ints, the Decimal must equal the *exact* binary value. We can verify this by converting the Decimal back to float — round-tripping should yield the original float exactly. For integers, the Decimal should equal the integer exactly.
5. **Sign preservation**: Negative inputs yield negative Decimals (including signed zero).

        # Property 5: integer inputs are represented exactly

        # Sign preservation (incl. signed zero via copysign)
