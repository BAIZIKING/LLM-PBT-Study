## Generation Strategy

To generate a wide variety of inputs and edge cases for `datetime.fromordinal(ordinal)`:

1. **Valid range coverage**: The documentation states valid ordinals are `1 <= ordinal <= datetime.max.toordinal()`. I'll use `st.integers()` with these explicit bounds to cover the full valid range.

2. **Edge cases**: Hypothesis automatically biases toward boundary values when given bounded integer strategies, so it will frequently test `1` (minimum), `datetime.max.toordinal()` (maximum), and values near these boundaries. This naturally exercises the January 1 of year 1 edge case and the maximum date.

3. **Wide distribution**: Integers across the whole range will produce datetimes spanning thousands of years.

## Properties to Check

Based on the API documentation, the returned datetime must satisfy:

1. **Hour, minute, second, microsecond are all 0** — explicitly stated.
2. **tzinfo is None** — explicitly stated.
3. **Round-trip consistency**: `result.toordinal()` should equal the input ordinal, since `fromordinal` is the inverse of `toordinal` (January 1 of year 1 has ordinal 1, establishing the correspondence).
4. **Return type**: The result must be a `datetime` instance.
