## Generation Strategy

To generate a wide variety of inputs for `Fernet.encrypt`, I focus on the `data` parameter, which must be `bytes`:

1. **Use `st.binary()`** to generate raw byte strings. This naturally covers many edge cases:
   - Empty bytes (`b""`)
   - Single-byte values
   - Long byte sequences
   - All possible byte values (0–255), including null bytes and high-value bytes
2. **Set a generous `max_size`** so we test both small and reasonably large payloads.
3. The Fernet key itself is generated fresh per test case via `Fernet.generate_key()`, ensuring the encryption object is always valid.

## Properties to Check

Based on the API documentation:

1. **Return type**: `encrypt` returns `bytes` (the Fernet token).
2. **URL-safe base64 encoding**: The token must be valid URL-safe base64. We can verify this by decoding it with `base64.urlsafe_b64decode` without error.
3. **Round-trip correctness (privacy/integrity)**: Decrypting the token with the same key must return exactly the original input data. This validates that the encryption is reversible and lossless.
4. **Non-emptiness / authenticity**: The token should be non-empty and verifiable (decryption succeeds without `InvalidToken`).
