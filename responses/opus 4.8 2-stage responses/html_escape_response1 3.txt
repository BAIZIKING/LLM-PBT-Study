1. The output never contains an unescaped `&`, `<`, or `>` character; every such character in the input is replaced by its corresponding HTML entity (`&amp;`, `&lt;`, `&gt;`).

2. When `quote` is true, the output contains no unescaped `"` or `'` characters; they are replaced by `&quot;` and `&#x27;` respectively. When `quote` is false, any `"` and `'` characters appear unchanged in the output.

3. If the input string contains none of the special characters (`&`, `<`, `>`, and also `"`/`'` when quote is true), the output is identical to the input (idempotent on "safe" strings).

4. The output is always a string, and an empty input string produces an empty output string.

5. All characters in the input that are not among the special characters being escaped appear unchanged and in the same relative order in the output.
