1. The output length always equals the length of the input array `x` (each input value maps to exactly one bin result).

2. When `labels=False`, the output is an ndarray of integer bin indicators, and every non-NA integer value falls within the range `[0, number_of_bins - 1]`.

3. Any NA value in the input `x`, as well as any value falling outside the range covered by the bins, produces an NA/NaN in the corresponding output position.

4. When `bins` is an integer `n` (and `duplicates='raise'`), the resulting Categorical has exactly `n` categories; and when `retbins=True`, the returned bins array has length `n + 1` and is monotonically increasing.

5. For each non-NA input value, the interval/bin assigned to it actually contains that value according to the `right` and `include_lowest` inclusivity rules (e.g., with `right=True`, a value v is placed in interval `(a, b]` such that `a < v <= b`).
