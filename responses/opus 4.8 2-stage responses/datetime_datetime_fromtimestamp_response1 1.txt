1. The return value is always a `datetime` instance.

2. When `tz` is None (or not specified), the returned `datetime` is naive (its `tzinfo` attribute is None).

3. When `tz` is a valid `tzinfo` instance, the returned `datetime` is aware and its `tzinfo` reflects/corresponds to the provided `tz`.

4. Converting the resulting `datetime` back to a timestamp (via `.timestamp()`) yields a value equal to the original input timestamp (modulo sub-second rounding and platform precision).

5. The `fold` attribute of the returned `datetime` is always either 0 or 1.
